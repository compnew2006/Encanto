# Requirements: QR Code Real-Time Delivery

## Overview
WhatsApp instances require scanning a QR code to pair. The QR code is generated by `whatsmeow` and stored in the DB, but never pushed to the frontend in real-time. Users must manually refresh to see an updated QR code.

## Requirements

### REQ-1: QR Code WebSocket Push
WHEN whatsmeow emits a new QR code event for an instance
THE SYSTEM SHALL immediately broadcast a WebSocket message of type `instance.qr_updated` to all clients in the same organization

### REQ-2: QR Code Payload
THE SYSTEM SHALL include in the `instance.qr_updated` payload:
- `instance_id`: UUID of the instance
- `qr_code`: the raw QR string for the frontend to render
- `expires_at`: ISO8601 timestamp 60 seconds from now

### REQ-3: Connected State Push
WHEN an instance successfully pairs and transitions to `connected`
THE SYSTEM SHALL broadcast `instance.status_changed` with `{ instance_id, status: "connected" }` to the organization

### REQ-4: Frontend QR Rendering
WHEN the frontend receives `instance.qr_updated`
THE SYSTEM SHALL render the QR code using a QR library (e.g., `qrcode` npm package)
AND show a countdown timer until expiry

### REQ-5: Auto-Close Modal
WHEN the frontend receives `instance.status_changed` with `status: "connected"` for the current instance
THE SYSTEM SHALL close the QR modal and show a success toast
