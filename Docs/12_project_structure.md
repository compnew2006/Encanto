# هيكل العمل العام للمشروع (Project Structure & Architecture)

الهدف من هذا الملف هو تحديد البنية الأساسية للمشروع بحيث تكون الحدود واضحة وصارمة بين: الواجهة، المنطق التشغيلي، البيانات، والتشغيل.

هذه البنية تعتمد على فصل الاهتمامات (Separation of Concerns) بحيث لا تتداخل الأجزاء المختلفة، مما يسهل النمو والتعديل مستقبلا. 

## 1. الواجهة (Interface / Presentation)
الواجهة هي كل ما يتصل بالمستخدم أو بأطراف خارجية. دورها الوحيد هو تلقي الطلبات (Inputs)، ترجمتها، وتمريرها للمنطق، ثم عرض الإجابات.
**يمنع منعا باتا وضع قواعد وعمليات تخص (Business Rules) في هذا القسم.**

**الأقسام:**
- **تطبيق المستخدم (Frontend Web App):** 
  - واجهات الويب المبنية بـ Svelte 5 و SvelteKit.
  - تشمل المكونات (Components)، الصفحات (Pages)، ومسار الواجهة (Router).
- **مستقبلات الـ API (Controllers / Handlers):** 
  - استقبال طلبات HTTP القادمة من الواجهة وتجهيز الردود (JSON/HTTP Responses).
  - تحتوي على تعريف المسارات (App Routes).
- **مستقبلات الوقت الفعلي (WebSocket Handlers):**
  - إدارة الاتصالات المباشرة واستقبال إشارات العميل (Events) وتوجيهها للمنطق.

## 2. المنطق (Logic / Business Domain)
المنطق هو القلب النابض للمشروع. هنا تعيش القواعد التشغيلية ومسارات العمل الأساسية (Workflows) مثل التعيين، الصلاحيات، وإدارة المحادثات.
**لا يعلم هذا القسم عن كيفية عرض الواجهة ولا ماهية قاعدة البيانات المستخدمة.**

**الأقسام:**
- **الخدمات الأساسية (Core Services):**
  - منطق المحادثات، تعيين المهام (Assignment)، حساب الحصص، وتطبيق شروط الترخيص (Limits/Quotas).
  - تحديد قوانين معالجة الرسائل و إنشاء حملات الإرسال (Campaigns).
- **نماذج البيانات والتحقق (Domain Models & Validators):**
  - القواعد المجردة للمنظمة، المستخدم، وكل المدخلات الصالحة.
  - التحقق من الصلاحيات وأحقية الإجراء (Permissions & Visibility checks).

## 3. البيانات (Data / Persistence)
هذه الطبقة مسؤولة حصرياً عن استرجاع وحفظ البيانات في قاعدة البيانات أو التخزين المؤقت.
هذه الطبقة لا تتخذ قرارات، بل تنفذ وتخزن ما يطلبه المنطق فقط.

**الأقسام:**
- **مستودعات البيانات (Repositories):**
  - إخفاء استعلامات قواعد البيانات (SQL / ORM).
  - أمثلة لوظائف هذه الطبقة: `FindUserById`، `SaveConversationEvent`.
- **التخزين المؤقت والذاكرة (Caching & State):**
  - إدارة حالة الاتصال والجلسات في الذاكرة (Memory Maps / Redis).
- **المحولات الخارجية (External API Adapters):**
  - مكتبات الدخول على خدمات خارجية (مثل Meta Cloud API).

## 4. التشغيل (Operation / Infrastructure & Background)
هذه الطبقة مسؤولة عن إبقاء النظام حياً، تشغيل المهام في الخلفية، ومراقبة أجزائه بجانب البيئة التحتية وتجهيز المتغيرات البيئية.

**الأقسام:**
- **وظائف الخلفية (Background Workers & Jobs):**
  - المهام المجدولة (CRONs)، طوابير الحملات المجدولة، وعمليات التنظيف (Cleanup tasks).
- **الإعدادات والبيئة (Configuration):**
  - إعدادات النظام الداخلية، متغيرات البيئة (Env Vars)، وإعدادات الترخيص الأساسية.
- **الاعتمادية والتدقيق (Reliability & Observability):**
  - تسجيل التغييرات التشغيلية (Audit Logging)، مراقبة الأداء، والإبلاغ عن الأخطاء (Errors).
- **نصوص التشغيل والنشر (Deployment):**
  - ملفات النشر مثل Docker, Scripts.

## قواعد منع التداخل (Strict Dependency Rules)

للحفاظ على مرونة وقابلية تطوير هذا النظام طالما أنه ينمو:
1. **اتجاه التبعية (Dependency Direction):** 
   - الواجهة تعتمد على المنطق. التشغيل يعتمد على المنطق والبيانات.
   - **المنطق لا يعتمد على الواجهة أبداً** ولا يعتمد بشكل مباشر على إطار عمل قاعدة البيانات (Database Driver) بل على واجهات موحدة (Interfaces).
2. **هياكل البيانات (DTOs):** تنقل البيانات بين هذه الطبقات الأربع على شكل كائنات بيانات مبسّطة خالية من تعقيدات واجهات الويب أو الـ ORM المحددة.
3. **التشغيل يستدعي ولا يقرر:** 
   - المهام الخلفية (Jobs) لا تنشئ قواعد عمل خاصة بها، بل تقوم بنداء الخدمات الموجودة في طبقة المنطق بشكل متكرر أو آلي.
4. **تجنب المنطق في الواجهة:**
   - الواجهة سواء (Frontend أو API Handlers) دورها الترجمة فقط. اتخاذ القرار برفض طلب أو تمريره يجب أن يحدث في طبقة "المنطق" حصرياً، لتجنب حدوث ثغرات عند استخدام مسارات دخول أخرى.

## الهيكل التنظيمي المقترح للمجلدات (Suggested Folder Structure)

هذه ترجمة حرفية للحدود الأربعة على شكل مجلدات عمل:

```text
/
├── frontend/             # 1. الواجهة: تطبيق الواجهة الرسومية (Svelte 5 / SvelteKit)
│
├── backend/
│   │
│   ├── api/              # 1. الواجهة: Controllers و مسارات الـ API
│   ├── ws/               # 1. الواجهة: استقبال الـ WebSocket
│   │
│   ├── core/             # 2. المنطق: خدمات البزنس (Services) وقواعد العمل
│   ├── models/           # 2. المنطق: أنواع البيانات والقواعد المجردة
│   │
│   ├── data/             # 3. البيانات: استعلامات التخزين و Repositories
│   ├── cache/            # 3. البيانات: إدارة الذاكرة والـ Redis
│   │
│   ├── workers/          # 4. التشغيل: مهام الخلفية والـ Cron Jobs
│   ├── config/           # 4. التشغيل: بيئة النظام والإعدادات
│   └── audit/            # 4. التشغيل: طبقة مراقبة السجلات والأداء
│
└── deploy/               # 4. التشغيل: تجهيزات النشر والبنية التحتية
```
