الوحدات المؤكدة في الواجهة الحية

المنصة والتنقل

  • شريط جنب رئيسي فيه Dashboard, Chat, Chatbot, Agent Analytics, Campaigns, Settings
  • مبدّل منظمة في الشريط الجانبي مع إنشاء منظمة جديدة وحذف المنظمة المحددة
  • قائمة مستخدم تتضمن التوافر، /profile، الثيم، اللغة، وتسجيل الخروج

مساحة المحادثة

  • تبويبات Assigned و Pending
  • مربع بحث، زر Add Contact، وفلتر بالمؤسسة ونوع المحادثة والتاغات
  • إجراءات القائمة: إخفاء المحادثة وحذفها لكل صف
  • مؤشرات Public، Pending، وعدد غير المقروء

التعاون داخل المحادثة

  • إسناد المحادثة لوكيل عبر Assign Contact
  • زر تثبيت Pin وزر إضافي من نوع user-x يحتاج ربط endpoint واضح لإعادة ضبط المسؤولية
  • لوحة Notes للملاحظات الداخلية
  • لوحة Contact Info مع Tags و Collaborators وبيانات عامة للجهة

الرسائل والوسائط

  • محرر فيه emoji وquick message وattachment وprint
  • وسائط بعمليات Print وDownload وDownload All
  • عمليات Revoke message وRetry sending مع إظهار سبب الفشل مثل instance not connected
  • قبل إرسال الرسالة النصية فقط، backend يحاكي typing event عبر whatsmeow لمدة تتناسب مع طول النص، مع استثناء الملفات والوسائط

الإشعارات والحالات

  • مركز إشعارات مع Mark all as read وعناصر من رسائل غير مقروءة وتنبيهات نظام
  • درج Statuses مع Add status وقائمة بالحالات حسب الجهة وinstance
  • زر Refresh وتحديثات فورية عبر WebSocket

وحدات مجاورة مؤكدة

  • وجود صفحة /chatbot في الشريط الجانبي
  • وجود صفحة /analytics/agents
  • وجود صفحة /campaigns
  • لوحة بيانات جهة الاتصال تشير صراحة إلى chatbot flow settings كمرجع لإعداد عرض البيانات

مركز الإعدادات

  • شجرة الإعدادات تحتوي General, Chatbot, WhatsApp, Contacts, Closed Chats, Canned Responses, Tags, Teams, Users, Roles, API Keys, Webhooks, Custom Actions, SSO, License
  • صفحة /settings نفسها تحتوي تبويبات General, Appearance, Chat, Notifications
  • تبويب Appearance يحتوي Color Mode وTheme Style مع presets مثل Twitter, Ocean Breeze, Soft Pop, Amber Minimal
  • تبويب Chat يحتوي Media Grouping Window, Sidebar Contact View, Chat Background, وإعدادات إظهار Print وDownload وربط Closed Chats
  • تبويب Notifications يحتوي Email Notifications, New Message Alerts, Notification Sound مع Play, وCampaign Updates
  • الإعدادات العامة تؤكد وجود جدولة Uploads Cleanup مع تشغيل يدوي عبر Run Cleanup Now، ويجب أن تبقى هذه الخاصية للأدمن فقط
  • لم تظهر أي خاصية إسناد داخل صفحة /settings نفسها؛ الإسناد بقي محصوراً في مساحة المحادثات

الترخيص داخل المنتج

  • صفحة /settings/license تعرض إدارة offline activation والحصص وحالة التجديد للـ deployment
  • الأزرار المؤكدة: Refresh, Copy HWID, Activate license, Refresh status
  • في التدقيق الحي بتاريخ 2026-04-19 ظهرت الحالة Disabled مع بطاقات Organizations, Users / Org, WA Endpoints / Org, Storage / Org, وSubscription Days
  • الواجهة تعرض HWID كاملاً مع Short ID وتطلب إرساله إلى الـ vendor للحصول على signed security key خاص بهذا الخادم
  • من الكود المحلي تأكد أن الترخيص مبني على Ed25519-signed JWT مرتبط بـ HWID hash ويحمل حدود organizations, users, WA endpoints, workers, وstorage
  • المسار /settings/license للأدمن فقط، ويوجد أيضاً مسار cleanup خاص عند تجاوز الحصة المرخصة

إدارة مثيلات واتساب

  • صفحة /settings/instances تعرض instances كبطاقات لا كجدول فقط
  • أزرار مؤكدة: Health Dashboard, Add Account, Edit instance name, Delete instance, Disconnect, Connect / Scan QR
  • كل بطاقة تعرض Connected / Disconnected، الهاتف، JID، وقياسات Uptime, Queue, Sent / Received, Error Rate
  • إعدادات تشغيل لكل instance: Auto-sync history, Auto-download incoming media, Call auto-reject, Auto campaign, Chat Close Rating Settings, Assigned Chat Reset, وChat Source Tag

تحليلات الوكلاء

  • صفحة /analytics/agents تحتوي فلاتر All Agents, All, This month, Any، وزر Export CSV
  • تحتوي بطاقات مؤشرات مثل Transfers Handled, Completed conversations, Active Conversations, Avg Resolution Time, Avg Queue Time, Break Time, وAverage Rating
  • تعرض أقسام Transfer Trends, Conversation Sources, Agent Comparison, وجدول Customer Ratings

  • زر رقم الهاتف داخل الجدول يفتح المحادثة مباشرة، ولم تظهر أي خاصية إسناد داخل شاشة التحليلات نفسها

  • الأزرار المؤكدة: Import/Export وAdd Contact، وعلى كل صف Open chat, Edit, Delete
  • نموذج الإنشاء يطلب Phone Number, Profile Name, وWhatsApp Instance، ونموذج التعديل يسمح بتحديث الاسم والرقم

إضافات مطلوبة على الخطة

الواجهة والتجربة

  • دعم i18n عبر ملفات JSON مع حفظ locale المختار في الإعدادات
  • الثيمات تكون مبنية على Tailwind مع Light / Dark / System
  • الشريط الجانبي يتمدد عند اقتراب الفأرة مع زر تثبيت للحالة
  • إرسال الملفات عبر drag & drop داخل composer مع معاينة أولية

الصلاحيات والرؤية

  • حالة المستخدم تصبح available أو unavailable أو busy بدل toggle ثنائي
  • كل خاصية يمكن ضبطها كـ create أو update أو delete أو read-only
  • التحكم حسب الدور في رؤية وإرسال المحادثات unclaimed
  • التحكم في رؤية كل جهات الاتصال أو جهات instances محددة أو أرقام هاتف مصرّح بها فقط

التكاملات

  • الإبقاء على إنشاء/حذف API Keys ضمن الخطة بشكل صريح
  • دعم إنشاء/حذف Webhooks لأي events مدعومة
  • كل webhook تدعم Webhook URL وSecret اختياري وCustom Headers اختيارية

الترخيص والتفعيل

  • التفعيل يجب أن يكون offline عبر signed security key وليس مكالمة SaaS مباشرة أثناء التشغيل
  • الـ token يجب أن يحمل license_id, license_family_id, revision, license_kind, tier, وlimits التشغيلية
  • الـ backend يخزن token مشفرة at rest، ويعيد بناء الحالة دورياً مع grace period وtime rollback tolerance
  • عند تجاوز الحصة يجب نقل المستخدمين إلى /license-cleanup والسماح فقط بعمليات الحذف والتنظيف اللازمة للعودة ضمن الحد
  • يوجد نظام vendor منفصل لإصدار ومراجعة الـ licenses مع Generate, Verify, Registry, وmetadata مثل customer_name
